# arxiv26-003 — Joint Encoding of KV-Cache Blocks

## Why it matters in this project

Joint Encoding matters because it compresses across requests/blocks while trying to preserve the standard cache structure.

## Architectural role

**Role**: block-fusion compression architecture for scalable serving.

This is different from token eviction and different from low-bit quantization. It treats similar KV blocks as candidates for shared encodings.

## Benchmarking structure

### Scenario

- high-concurrency serving
- cross-request / cross-block redundancy

### Workload shape

- concurrent autoregressive inference
- scalable serving where rigid heuristics become limiting

### Metrics

- KV compression ratio
- accuracy retention
- scalability under concurrent serving
- comparison to structured/adaptive compression baselines

## Comparison use

Compare Joint Encoding with:

- KVTC
- ChunkKV
- PolarQuant

The key axis is whether the method exploits **cross-block redundancy** rather than only local token importance.

## Project takeaway

Joint Encoding pushes the project toward a more global view:

**some KV compression architectures exploit redundancy across requests, not only within one cache.**
