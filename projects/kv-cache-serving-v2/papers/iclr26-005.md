# iclr26-005 — TRIM-KV / Cache What Lasts

## Why it matters in this project

TRIM-KV is important because it represents the eviction/retention branch in a more learned and benchmark-diverse way than many heuristic methods. It is a good architectural counterpoint to retrieval systems.

## Architectural role

**Role**: learned retention architecture for memory-bounded KV serving.

Unlike offloading/retrieval papers, TRIM-KV does not ask “what should I fetch back?” It asks:

- what should never be evicted in the first place?
- can the system learn token utility at creation time?

Architecturally, it is still a memory-bounded design, but it places the main intelligence at token retention time rather than at retrieval time.

## What architectural problem it solves

The main problem is that heuristic eviction often relies on weak importance proxies, while retrieval/offloading approaches can incur substantial orchestration overhead.

TRIM-KV proposes a different architectural answer:

- keep the active cache bounded
- learn which tokens are worth preserving long-term
- avoid paying a large retrieval/control-path penalty later

## Benchmarking structure

This paper is benchmarked as a **memory-bounded retention/eviction** paper across unusually diverse workloads.

### Scenario

- fixed or constrained KV memory budget
- long-horizon inference under memory pressure

### Workload shape

- mathematical reasoning
- procedural generation
- conversational long-memory
- general long-context understanding

This is broader than many system papers, which is a strength but also means the workloads are not identical to production serving traces.

### Metrics

- task accuracy under memory budgets
- low-memory-regime robustness
- comparison to eviction and learnable retrieval baselines

The main value is **quality under bounded memory**, not deployment throughput alone.

### Why this matters for later comparison

TRIM-KV should be compared to:

- Ada-KV
- KeyDiff
- other eviction/retention methods
- retrieval papers only cautiously

If compared to retrieval papers, the right framing is:

- retention-first versus retrieval-first architectures

not simply “which one is faster.”

## Placement in the design space

This paper belongs to:

- memory-bounded inference
- learned retention / eviction
- quality-oriented long-horizon evaluation

## Project takeaway

TRIM-KV is useful because it sharpens a structural distinction for the project:

**some systems solve KV pressure by moving the right state back in; others solve it by never dropping the wrong state.**
