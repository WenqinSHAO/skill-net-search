# osdi24-002 — DistServe

## Why it matters in this project

DistServe is one of the clearest papers in the project for showing that KV cache pressure is not only a memory issue. It changes cluster architecture by splitting prefill and decode into different resource domains.

## Architectural role

**Role**: phase-disaggregated serving architecture.

The architectural move is not primarily compression or retrieval. It is decomposition:

- prefill is treated as one resource regime
- decode is treated as another resource regime
- KV transfer becomes the coupling interface between the two

This is a strong architectural step because it introduces a clean systems boundary around KV production and KV consumption.

## What architectural problem it solves

The core problem is phase interference:

- prefill and decode have different latency objectives
- colocating them couples scheduling and resource allocation
- serving systems then struggle to satisfy both TTFT and TPOT well

DistServe solves this by making phase separation explicit and then optimizing around the KV transfer boundary.

## Placement in the design space

This paper belongs to:

- phase-level disaggregation
- SLO-aware serving architecture
- KV transfer as a cluster interface

It is not trying to minimize KV size directly. Instead, it treats KV as the state object that must cross the prefill/decode boundary efficiently.

## Architectural consequence

DistServe teaches an important lesson for this project:

**KV architecture is also execution architecture.**

Once prefill and decode are separated, the system can:

- tune each phase independently
- place them differently
- optimize their resource plans separately

But this comes at the cost of a new systems burden:

- KV handoff is now unavoidable and central

## How to compare it to later papers

DistServe is a key comparison point for Mooncake:

- DistServe: phase disaggregation as the main architectural move
- Mooncake: phase disaggregation plus a fuller KV-centric storage/control architecture

It is also useful as a contrast with PagedAttention:

- PagedAttention optimizes inside a node
- DistServe changes the cluster-level decomposition

## Project takeaway

DistServe is one of the papers that turns KV from a memory object into an **inter-stage interface**. That is why it belongs near the center of the project’s architectural narrative.
