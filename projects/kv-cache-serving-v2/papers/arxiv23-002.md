# arxiv23-002 — CacheGen

## Why it matters in this project

CacheGen is an early anchor for the idea that reusable KV state is only useful if it can be moved cheaply enough to matter in end-to-end serving.

## Architectural role

**Role**: network-efficient KV serialization / streaming layer.

It is not mainly about local eviction or cluster scheduling. Its architectural contribution is to make reusable KV practical across network boundaries by compressing and streaming it.

## Benchmarking structure

### Scenario

- context reuse across requests
- network transfer of reusable KV state

### Workload shape

- long-context serving where context load latency matters
- bandwidth-constrained settings

### Metrics

- context loading delay
- bandwidth usage
- compression effectiveness with low decode overhead

## Comparison use

Compare CacheGen with later KV reuse systems when the question is:

- does the system assume KV is cheap to move?
- if not, what representation/transport layer makes reuse viable?

## Project takeaway

CacheGen is one of the earliest papers showing that KV reuse is also a **transport problem**, not only a cache-hit problem.
