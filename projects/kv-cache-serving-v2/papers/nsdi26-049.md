# nsdi26-049 — Seshat

## Why it matters in this project

Seshat matters because it gives workload characterization and workload generation for production LLM serving, which is essential for evaluating KV-serving proposals realistically.

## Architectural role

**Role**: workload characterization and trace-generation reference.

It is not a KV mechanism paper. Its architectural value is evaluative: it helps define what realistic serving workloads look like.

## Benchmarking structure

### Scenario

- worldwide cloud inference serving service

### Workload shape

- language models
- multimodal models
- reasoning models
- production diversity rather than one synthetic workload family

### Metrics

- workload distributions and correlations
- realism of generated traces
- representativeness for benchmarking serving systems

## Comparison use

Use Seshat to judge later benchmark credibility:

- does a paper’s evaluation workload resemble production serving structure at all?

## Project takeaway

Seshat is critical because later cross-paper comparison depends on **benchmark realism**, not just mechanism novelty.
