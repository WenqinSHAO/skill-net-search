# iclr26-001 — KVTC

## Why it matters in this project

KVTC is an important storage/compression anchor because it treats KV compression as a general coding problem rather than only heuristic token pruning.

## Architectural role

**Role**: transform-coding layer for compact KV storage.

It is especially relevant where reusable or stale KV must remain available on-GPU or off-GPU without forcing recomputation.

## Benchmarking structure

### Scenario

- reusable KV caches across conversation turns and iterative workflows
- compact on-GPU and off-GPU storage

### Workload shape

- reasoning and long-context benchmarks
- storage efficiency without modifying model parameters

### Metrics

- compression ratio
- reasoning and long-context accuracy retention
- comparison against eviction, quantization, and SVD-style baselines

## Comparison use

Compare KVTC with:

- PolarQuant
- ChunkKV
- Joint Encoding

The useful axis is whether the method is:

- token-selective
- quantization-oriented
- transform/coding-oriented

## Project takeaway

KVTC represents the branch where KV efficiency is treated as a **signal coding problem for reusable state**.
