# nsdi26-038 — FlexLLM

## Why it matters in this project

FlexLLM matters because it broadens the resource-sharing context around inference serving. It shows how inference and finetuning may contend for the same system resources.

## Architectural role

**Role**: co-serving architecture for inference and finetuning.

It is not a direct KV retrieval or reuse paper, but it affects the project by redefining the resource environment in which KV-heavy inference runs.

## Benchmarking structure

### Scenario

- shared GPUs for inference and PEFT-based finetuning

### Workload shape

- mixed inference and training tokens
- latency-sensitive inference under co-serving conditions

### Metrics

- inference SLO compliance
- utilization
- memory savings
- overall co-serving efficiency

## Project takeaway

FlexLLM matters because it reminds us that KV-serving designs live inside broader cluster-sharing architectures, not in isolation.
