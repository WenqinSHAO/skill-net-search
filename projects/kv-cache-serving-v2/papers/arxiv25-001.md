# arxiv25-001 — LMCache

## Why it matters in this project

LMCache is a central reuse-system paper because it turns KV into a shareable layer outside GPU memory and across inference engines.

## Architectural role

**Role**: external KV cache layer across queries and engines.

Architecturally, LMCache promotes KV from engine-internal state to a reusable service layer:

- offloaded outside GPU
- reused across queries
- shared across engines such as vLLM and SGLang

## Benchmarking structure

### Scenario

- enterprise-scale LLM inference
- prefix reuse and PD disaggregation support

### Workload shape

- cross-query prefix reuse
- cross-engine / cross-GPU transfer
- growing persistent KV footprint

### Metrics

- reuse efficiency
- offload/transfer overhead
- serving efficiency across engines

## Comparison use

Compare LMCache with:

- CacheGen
- HCache
- KVShare

The core comparison axis is whether the architecture makes KV into a reusable external service layer.

## Project takeaway

LMCache is one of the papers that most clearly says: **KV is becoming infrastructure, not only memory.**
