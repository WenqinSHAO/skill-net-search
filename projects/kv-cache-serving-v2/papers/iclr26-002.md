# iclr26-002 — IceCache

## Why it matters in this project

IceCache is important because it sits in the retrieval/offloading branch, but with a more structured memory view than naive token selection. It is useful as a bridge between generic offloading and later finer-grained retrieval systems.

## Architectural role

**Role**: semantically clustered KV retrieval for offloading-based long-context serving.

Architecturally, IceCache assumes KV does not fit on GPU and that retrieval quality matters. Its move is to organize semantically related tokens into contiguous regions so that transfer and selection become more structured.

## What architectural problem it solves

The underlying problem is a double inefficiency:

- retrieving too much KV wastes bandwidth and latency
- retrieving the wrong KV hurts long-context accuracy

IceCache addresses this by adding a clustering/data-structure layer between full-cache offloading and on-demand retrieval.

## Benchmarking structure

This paper is benchmarked as a **long-context retrieval/offloading** paper.

### Scenario

- long-context LLM inference
- CPU-GPU KV offloading
- constrained on-GPU KV budget

### Workload shape

- long-context benchmarks, especially long-generation and reasoning-sensitive settings
- quality-sensitive retrieval under tight token budgets

### Metrics

- accuracy retention versus full-cache baseline
- latency against offloading-based baselines
- KV token budget efficiency

The key reported structure is not just a speedup claim. It is a **quality-at-budget** claim, for example retaining near-full accuracy under a small token budget.

### Why this matters for later comparison

IceCache should be compared to:

- InfiniGen
- LouisKV
- PRKV

The right axis is:

- retrieval unit granularity
- transfer efficiency
- quality under constrained KV budget

Not all of these systems optimize the same latency target, so raw speedup alone is not a fair comparison.

## Placement in the design space

This paper belongs to:

- offloading-based KV serving
- structured retrieval
- long-context accuracy preservation

## Project takeaway

IceCache shows that retrieval architecture is not only about moving less data. It is about building a structure that preserves the **useful semantic working set** under tight GPU budgets.
