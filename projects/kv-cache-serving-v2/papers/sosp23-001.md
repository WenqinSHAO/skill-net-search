# sosp23-001 — PagedAttention / vLLM

## Why it matters in this project

This paper is the architectural baseline for the whole project. It establishes that KV cache is not just a tensor footprint problem, but a memory-management problem that shapes serving throughput.

Without PagedAttention, later papers on disaggregation, retrieval, reuse, or sharing are harder to interpret, because the field has not yet cleanly separated:

- model compute
- serving scheduler behavior
- KV memory layout and allocation

## Architectural role

**Role**: local memory virtualization layer for KV state.

Architecturally, PagedAttention does not yet disaggregate KV across machines or storage tiers. Instead, it changes the abstraction boundary inside a single serving node:

- KV cache becomes page-managed state rather than monolithic contiguous state
- fragmentation becomes a first-class systems concern
- cache sharing becomes possible because the representation is more flexible

This is the moment where KV cache starts to look like an allocatable systems resource.

## What architectural problem it solves

The key architectural bottleneck is not simply “KV is large.” It is:

- KV grows dynamically per request
- allocation is irregular
- waste from fragmentation reduces usable batch capacity

PagedAttention reframes the serving architecture from “fit requests into memory” to “manage a virtualized KV address space efficiently.”

## Placement in the design space

This paper belongs to:

- single-node serving architecture
- local KV memory management
- memory layout / allocation primitive

It is not yet:

- a disaggregated system
- a retrieval system
- a cross-model sharing system

## Architectural consequence

This paper creates the base layer that many later papers implicitly build on:

1. KV can be treated as a managed object rather than a passive byproduct.
2. Better memory layout directly unlocks better batching and throughput.
3. Sharing and reuse become more practical once KV is addressable and composable.

## How to compare it to later papers

Use this paper as the baseline for the question:

“Is the later paper still optimizing local KV residency, or has it moved the architectural boundary outward?”

Examples:

- DistServe moves the boundary to phase-level resource separation.
- Mooncake moves the boundary to multi-tier disaggregated KV storage.
- DroidSpeak moves the boundary to cross-model KV reuse.

## Project takeaway

PagedAttention is not the final architecture in this project; it is the foundational substrate. It tells us the first durable lesson of the field:

**KV-aware serving starts when KV layout becomes a systems abstraction.**
