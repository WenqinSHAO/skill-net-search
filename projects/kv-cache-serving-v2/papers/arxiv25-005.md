# arxiv25-005 — SemShareKV

## Why it matters in this project

SemShareKV matters because it pushes reuse from lexical similarity toward semantic similarity.

## Architectural role

**Role**: semantic KV sharing architecture via fuzzy matching.

Rather than requiring identical prefixes, it uses token-level LSH matching and positional handling to recover reusable state from semantically similar prompts.

## Benchmarking structure

### Scenario

- prompts that are semantically similar but not token-identical
- summarization and conversational workloads

### Workload shape

- semantic overlap with lexical variation
- reuse quality depends on approximate matching fidelity

### Metrics

- speedup
- GPU memory reduction
- output quality degradation

## Comparison use

Compare SemShareKV with:

- KVShare
- LMCache
- DroidSpeak

The key distinction is semantic approximate reuse versus exact or architecture-compatible reuse.

## Project takeaway

SemShareKV shows that reuse scope can be widened if the system is willing to treat KV sharing as a **similarity search problem**.
