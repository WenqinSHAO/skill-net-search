# atc25-001 — KVCache Cache in the Wild

## Why it matters in this project

This paper is important because it is one of the few workload-characterization papers focused specifically on KV cache behavior in production cloud serving.

## Architectural role

**Role**: workload-grounding and eviction-policy design reference.

It does not introduce a fundamentally new KV mechanism. Instead, it tells the project how real reuse patterns behave and why workload-aware cache policy matters.

## Benchmarking structure

### Scenario

- large cloud-provider serving traces
- production KV cache reuse

### Workload shape

- skewed reuse across requests
- both single-turn and multi-turn reuse matter
- predictability differs by request category

### Metrics

- cache hit behavior
- throughput / latency under real traces
- policy effectiveness under limited cache capacity

## Comparison use

This paper should be used to sanity-check later design claims:

- does a proposed eviction or reuse method assume a workload pattern that production traces actually support?

## Project takeaway

This paper gives the project a necessary reminder:

**KV-serving architecture is workload-shaped; synthetic assumptions can mislead cache-policy design.**
