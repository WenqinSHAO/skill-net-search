# arxiv25-008 — SamKV

## Why it matters in this project

SamKV is central to the new project because it directly addresses reuse outside the single-prefix, single-context setting.

## Reuse-contract view

**Role**: sparse reuse over multiple-context KV caches.

In RAG-like settings, each retrieved document may have its own independently computed KV cache. SamKV asks how sparse attention can select and locally recompute across this multiple-context retained state without keeping or fully recomputing everything.

## Beyond-prefix angle

This paper is one of the clearest ways the project goes beyond prefix reuse:

- the reusable object is no longer one causal prefix stream
- it is a set of independently computed context caches
- reuse is widened by sparse cross-context selection plus local recomputation

## Storage-versus-computation trade

SamKV makes the trade explicit:

- retaining all multiple-context KV caches is expensive
- recomputing all cross-context interactions is also expensive
- sparse selection plus local recomputation can sit in between

This is highly relevant for the project because it looks like a realistic path for reusable state in retrieval-augmented or tool-augmented systems.

## Comparison use

Compare SamKV with:

- `KVShare`
- `CentroidKV`
- `NOSA`

The important question is whether the system widens reuse by:

- approximate request-level matching
- clustering within KV state
- sparse selection across multiple retained contexts
- sparse attention with native offloadability
