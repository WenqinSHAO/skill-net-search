# arxiv26-004 — HySparse

## Why it matters in this project

HySparse is one of the strongest additions for the project’s future section because it directly connects sparse attention and KV cache sharing inside the model architecture.

## Reuse-contract view

**Role**: intra-model KV sharing between full and sparse attention layers.

HySparse does not mainly reuse KV across requests. It reuses KV across layers and attention regimes within one execution graph.

That matters because it broadens the meaning of reuse:

- not only reuse across time
- but reuse across architectural components

## Beyond-prefix angle

HySparse is relevant because it suggests that the field may move beyond prefix reuse by redesigning the model so that:

- sparse layers can inherit or share the KV generated by nearby full-attention layers
- token selection and KV reuse become co-designed

This is an especially important future-facing signal for the project.

## Storage-versus-computation trade

HySparse argues that a better trade can come from model architecture itself:

- fewer full-attention layers
- reused KV across sparse layers
- lower stored KV and lower computation together

That is different from service-layer reuse papers, but highly relevant to the project because it changes whether reusable KV is cheap enough to keep.

## Comparison use

Compare HySparse with:

- `NOSA`
- `SamKV`
- `DroidSpeak`

The important distinction is whether reuse happens:

- within one model architecture across layers
- across adjacent sparse-attention decoding steps
- across multiple contexts
- across different model executions
