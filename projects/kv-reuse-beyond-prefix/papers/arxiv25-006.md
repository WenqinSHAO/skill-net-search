# arxiv25-006 — PRKV

## Why it matters in this project

PRKV matters here because it is not just a generic retrieval paper. It helps define a middle ground between full-cache preservation and crude page-level retrieval by restructuring the retrieval unit itself.

## Reuse-contract view

**Role**: structured selective reuse under offloaded KV.

PRKV still assumes the full KV state is retained off device, but it tries to make that retained state more reusable at decode time by improving the page-level access structure.

In this project’s terms, it is important because it asks:

- can a system retain full historical state cheaply enough off device
- while still reusing only the right parts with acceptable overhead?

## Beyond-prefix angle

PRKV does not go beyond prefix reuse in the semantic or cross-model sense.

Its importance is different:

- it makes retained KV more efficiently reusable when exact full-history access is too expensive
- it suggests that the reuse contract can also be widened by restructuring the access unit, not only by changing similarity or compatibility logic

## Storage-versus-computation trade

PRKV is useful for the project because it makes the trade more explicit:

- storage cost stays high because the complete KV is still retained
- compute saved can still be high because the system avoids recomputation
- the decisive factor becomes retrieval overhead and access granularity

## Comparison use

Compare PRKV with:

- `CentroidKV`
- `SamKV`
- `NOSA`

The key contrast is whether the system improves reuse by:

- better retained-state access structure
- clustering and merging
- sparse attention over multiple contexts
- sparse-attention locality that makes offloading easier
