# arxiv25-010 — NOSA

## Why it matters in this project

NOSA matters because it tests one of the project’s central hypotheses: sparse attention may not only reduce computation, it may also make retained KV more practical to offload and reuse.

## Reuse-contract view

**Role**: sparse-attention architecture that makes KV offloading more natively reusable.

Its key move is to impose explicit locality constraints so that the selected KV pairs become easier to move and reuse across adjacent decoding steps.

## Beyond-prefix angle

NOSA does not widen reuse by semantic matching or cross-model compatibility.

Its relevance is more infrastructural:

- it changes the attention pattern so retained KV becomes more offloadable
- in doing so, it may make reusable state cheaper even if the compatibility contract stays unchanged

## Storage-versus-computation trade

NOSA is useful because it suggests a different route to a better trade:

- not only save compute directly
- but reduce transfer burden enough that keeping reusable state off device becomes more attractive

This is important for the project’s future-looking part, because it links attention design directly to reusable-state economics.

## Comparison use

Compare NOSA with:

- `HySparse`
- `SamKV`
- `LMCache`

The key distinction is whether the system improves reuse by:

- making sparse attention natively offloadable
- sharing KV between full and sparse layers
- sparsifying multiple-context retained state
- externalizing reusable KV as infrastructure
